Dist_SMA200_D1 sign, meta-classifier sur 7 features). Les résultats OOS sur

veux SAVOIR si l'edge gagné est statistiquement réel.

TÂCHE : Créer un module learning_machine_learning/analysis/edge_validation.py 
qui implémente trois tests :

1. **Breakeven WR vs WR observé** : 
   - Calculer breakeven_wr = (sl + friction) / (sl + tp) pour la config courante 
     (BacktestConfig.tp_pips, sl_pips, commission_pips, slippage_pips)
   - Comparer au WR observé sur 2024 et 2025
   - Marge = wr_observé - breakeven_wr (négatif = perdant structurel)

2. **Bootstrap Sharpe ratio** :
   - Resampler les trades avec remise (10000 iterations) pour générer une 
     distribution du Sharpe
   - Calculer p-value(Sharpe > 0) = % d'iterations où Sharpe > 0
   - Confidence interval 95%
   - À faire par année OOS séparément

3. **Deflated Sharpe Ratio (López de Prado, ch.14)** :
   - Corrige le Sharpe observé pour le nombre de trials testés (data-snooping)
   - DSR = (SR_observed - E[SR_max]) / σ(SR)
   - Où E[SR_max] dépend de n_trials et de la skewness/kurtosis des returns
   - Reporter PSR (Probabilistic Sharpe Ratio) aussi

EXTRA : ajouter un test t-statistique simple sur l'expectancy par trade 
(t = mean(pnl) / (std(pnl) / sqrt(n)) avec hypothèse nulle E[pnl]=0).

LIVRABLE :

CRITÈRE DE DÉCISION POUR LA SUITE :
- p-value bootstrap(Sharpe > 0) < 0.05 sur 2024 ET 2025 → edge réel, passer 
  à walk-forward
- p-value > 0.10 sur 2025 → edge non confirmé, NE PAS continuer les 
  optimisations cosmétiques, restructurer le problème (changer TF, 
  instrument, ou cible)
- 0.05 < p < 0.10 → zone grise, faire D + E + F en parallèle
