J'ai une stratégie ML de trading EURUSD H1 (triple barrier TP=20p / SL=10p / 
window=24h) dans /d:/Documents/learning-machine-learning. Le pipeline a 4 
scripts (1_clean_data → 2_master_feature_engineering → 3_model_training → 
4_backtest_triple_barrier), un modèle RandomForest train ≤ 2023 avec embargo 
48h, split 3-étages strict (2024 VAL, 2025 TEST).

DIAGNOSTIC ACTUEL : après audit complet, le modèle n'a pas d'edge prédictif 
réel en OOS. Sur 600+ trades cumulés 2024-2025, le WR est ~25% dans LES DEUX 
directions LONG et SHORT, alors que le breakeven WR (TP=20/SL=10/friction 1.5p) 
est 38.3%. WR LONG 2025 = 25.8% dans un marché qui monte de +14% = pire que 
random. Les Sharpe in-sample 7-9 sur 2022-2023 étaient du surapprentissage 
massif révélé une fois les bugs corrigés.

Pistes cosmétiques épuisées : filtres trend/vol/session fonctionnels (filtre 
tendance était silencieusement désactivé, maintenant fixé avec Dist_SMA200_D1 
+ raise si feature absente), features pourries droppées (Dist_SMA200_D1, 
ATR_Norm, Volatilite_Realisee_24h gardées comme features-filtre mais exclues 
du training), 3 variantes TP/SL testées (toutes perdent), 2 seuils testés 
(0.38 → 4 trades, 0.35 → 350 trades, mêmes conclusions).

DEUX PISTES RESTANTES À TRANCHER :

Piste B — Walk-forward retraining : réentraîner glissant tous les 3-6 mois. 
Probabilité de succès ~30%. Coût 1 jour. Argument contre : même 2024 (proche 
du cutoff train) perd à 25% WR, donc la non-stationnarité n'est probablement 
pas la cause.

Piste C — Cible binaire + meta-labeling (López de Prado, Advances in 
Financial Machine Learning ch.3) : modèle 1 = direction simple (ex SMA cross 
ou momentum), modèle 2 = classifieur binaire "ce trade va-t-il gagner". Le 
seuil opère sur le 2ème modèle. Probabilité de succès ~50%. Coût 2-3 jours. 
Conçu exactement pour les signaux faibles comme le mien.
